## Requirements:

- User must be able to identify foods on camera/galery photos with built model.

- User must be able to see information about identified foods. Information will include food's name, calorie for 1 portion, a simple recipe and can include special comment.

- User must be able to search any food's page manually with filters.

- User must be able to see all daily intake lists.

- Daily intake lists will have foods, food's calorie and portion; total calorie eaten information.

- User must be able to add/remove foods they have eaten to their daily food intake list.

- User must be able to see total calorie intake, eaten foods etc. statistic graphs. Users must be able to filter these statistics by time.

## Technologies:

- Flutter (To qucikly make a mobile app that can work on both android and ios)

- Python server (To deploy the model for identfying foods)

- SQLite (For storing food's information and daily intake lists)

- VSCode ide (Experience)
